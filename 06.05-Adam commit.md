W api.py dodano obsługę błędu:

Ten błąd jest mapowany na HTTP 400. Efekt: jeśli ktoś próbuje załadować ten sam raw_file_id drugi raz do tego samego stagingu, API zwraca kontrolowany błąd, a nie 500.
